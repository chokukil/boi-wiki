업무 흐름 정의

BoI Wiki에서 기존 SOP, Event, Action, Skill을 먼저 찾고 재사용합니다. 부족한 항목만 draft로 보완해 업무 흐름 정의에 연결합니다.

추가 시작

새 API, MCP, Skill, Event, Action을 따로따로 만들기 전에 먼저 어떤 업무 BoI를 채워야 하는지를 확인합니다. 업무 흐름 정의는 그 업무 목적과 근거, 다음 행동, 완결 조건을 묶는 내부 실행 정의입니다.

SOP가 있는 정형 업무는 SOP 실행 흐름으로 정렬하고, SOP가 없는 반복/비정형 업무는 업무 BoI와 처리 패턴을 기준으로 초안을 만들 수 있습니다.

업무 목적 업무 BoI 근거 다음 행동 완결 조건

추가 / 연결 시작

이 화면의 추가는 새 항목을 바로 만드는 버튼이 아니라, 기존 항목을 찾고 재사용할지 확장할지 결정하는 시작점입니다.

{% if start %}시작 위치: {{ start }}{% endif %}

1. 어떤 업무를 완성하나요?

업무 목적, 채워야 할 업무 BoI, 필요한 근거, 완결 조건을 먼저 적습니다.

2. 기존 항목을 재사용하나요?

SOP, Event 정의, Action, Skill/Connector, 기존 업무 흐름 정의를 중복 검색합니다.

3. 부족한 항목만 draft로 만듭니다

없는 Event, Action, SOP, Skill은 바로 운영 반영하지 않고 검증 가능한 초안으로 남깁니다.

4. 검증 후 publish합니다

schema, smoke, 권한, approval을 통과한 항목만 BoI Wiki에 반영합니다.
